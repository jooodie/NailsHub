# 台南美甲預約網站平台 MVP 規格草案

## 1. 專案概述

### 專案名稱
台南美甲預約網站平台

### 專案目標
打造一個面向台南地區美甲店家與顧客的線上預約平台，讓店家能夠管理服務項目與可預約時段，讓顧客能夠瀏覽店家資訊、查看服務價格並完成預約。未來可延伸串接金流、訂金機制、評價系統與平台抽成模式。

### MVP 核心驗證問題
1. 顧客是否願意透過平台完成美甲預約，而不是透過 Instagram 或 LINE 私訊。
2. 店家是否願意使用平台管理服務內容、時段與預約狀態。
3. 平台是否能有效降低手動排程與撞單問題。

---

Runtime：Python 3.11+
Web：FastAPI + Uvicorn（或 Gunicorn + Uvicorn workers 上線用）
DB：PostgreSQL
ORM／遷移：SQLAlchemy 2.x + Alembic
設定：pydantic-settings（.env 分環境）
API 風格：REST、/shops、/shops/{id}/stylists 這類資源路徑；錯誤格式統一即可


## 2. 目標使用者

### 使用者類型

#### A. 顧客
- 想在台南尋找美甲店
- 希望快速查看服務、價格與可預約時段
- 不想透過私訊來回確認時間

#### B. 美甲店家 / 美甲師
- 想展示自己的店面與作品
- 想管理服務項目與價格
- 想簡化預約流程，降低漏單與改約成本

#### C. 平台管理者（Admin）
- 管理店家上架資訊
- 查看平台上的預約紀錄
- 維護平台基本營運秩序

---

## 3. 產品定位

本平台是一個以「線上預約」為核心的雙邊平台。

### MVP 主要提供的價值

#### 對顧客
- 可直接查看台南美甲店家資訊
- 可快速查看服務項目、價格、時段
- 可線上完成預約

#### 對店家
- 可建立自己的店家頁面
- 可管理服務項目與可預約時間
- 可在後台查看與更新預約狀態

---

## 4. MVP 功能範圍

### 4.1 顧客端功能

#### 1. 首頁 / 店家列表
- 顯示台南地區美甲店家列表
- 顯示店家名稱、封面圖片、區域、簡短介紹
- 支援基本篩選（例如：台南行政區）

#### 2. 店家詳細頁
- 顯示店家介紹
- 顯示地址 / 區域
- 顯示服務項目與價格
- 顯示可預約時段
- 提供預約按鈕

#### 3. 預約功能
- 選擇服務項目
- 選擇日期與時段
- 填寫基本資料（姓名、電話、備註）
- 送出預約申請

#### 4. 預約結果頁
- 顯示預約成功
- 顯示店家名稱、服務項目、預約時間、顧客資訊

---

### 4.2 店家端功能

#### 1. 店家登入
- 店家可使用帳號密碼登入後台

#### 2. 店家資料管理
- 店名
- 店家介紹
- 地址
- 台南行政區
- 聯絡方式
- 營業時間
- 店家封面圖 / 作品圖

#### 3. 服務管理
- 新增服務項目
- 編輯服務項目
- 刪除服務項目
- 設定價格
- 設定服務時長
- 設定服務說明

#### 4. 可預約時段管理
- 設定每週固定可預約時段
- 手動關閉特定日期或特定時段
- 避免重複預約

#### 5. 預約管理
- 查看預約列表
- 查看預約詳細資訊
- 更新預約狀態：
  - 待確認
  - 已確認
  - 已完成
  - 已取消

---

### 4.3 Admin 功能

#### 1. 店家管理
- 查看全部店家資料
- 審核店家是否上架

#### 2. 預約管理
- 查看全平台預約紀錄

---

## 5. MVP 不包含的功能

以下功能暫不納入 MVP，以避免開發範圍過大：
- 即時聊天功能
- 顧客評價與評論系統
- 優惠券與折扣機制
- 收藏店家
- 推薦演算法
- Google / LINE 第三方登入
- 簡訊通知
- 平台抽成結算
- 完整線上金流付款
- 多分店與多員工複雜排班

## 6. 金流相關規劃（先預留，不一定在 MVP 上線）

雖然 MVP 可先不串接實際金流，但資料結構與流程建議先預留以下欄位：
- total_amount
- payment_status

## 7. 使用者流程（User Flow）

### 顧客流程
1. 進入首頁
2. 瀏覽台南美甲店家列表
3. 點擊進入店家詳細頁
4. 查看服務項目、價格與可預約時段
5. 選擇服務與時段
6. 填寫個人資料並送出預約
7. 顯示預約成功頁面

### 店家流程
1. 店家登入後台
2. 建立或編輯店家資料
3. 新增服務項目
4. 設定可預約時段
5. 查看顧客預約
6. 更新預約狀態

### Admin 流程
1. 管理員登入
2. 查看待審核店家
3. 管理平台店家與預約資料

## 11. MVP 驗收條件

### 功能面
1. 顧客可以成功瀏覽店家列表與店家詳細頁。
2. 顧客可以成功送出一筆預約。
3. 店家可以在後台新增服務項目與設定時段。
4. 店家可以查看與更新預約狀態。
5. 系統可避免同時段被重複預約。

### 體驗面
1. 顧客能在 3 分鐘內完成一筆預約。
2. 店家能在 10 分鐘內完成店家資料與服務上架。

## 13. 一句話總結

這個 MVP 的目標不是一次做成完整商業平台，而是先做出一個可驗證的預約系統，確認顧客與店家是否真的會使用平台完成預約與管理流程。
